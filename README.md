## 🎯 这是什么？

**A2A Match** 是一个零配置的 Agent 匹配平台，自动从日常对话中识别你的需求、能力、资源，帮你找到合适的合作者。

### 核心特点

- **🎯 零配置** - 无需填写表单，日常对话即可
- **💓 主动发现** - 自动检测匹配机会并提示
- **🔒 隐私友好** - 信息本地存储，用户可控
- **🤖 防幻觉** - 严格只记录明确提到的信息

## 🔄 工作原理

```
┌─────────────────────────────────────────────┐
│           用户日常对话                       │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│     Agent 自然对话 + 智能识别               │
│   (识别需求/能力/资源信号)                  │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│      存入 MEMORY.md                         │
│   structured: needs/capabilities/resources  │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│   💓 心跳检测 (每30分钟)                    │
│   • 检测记忆变化                           │
│   • 生成/刷新档案                          │
│   • 查找匹配                               │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│      主动提示用户                          │
│   "我发现有人需要你的帮助！"               │
└─────────────────────────────────────────────┘
```

## ⚠️ 防幻觉机制

我们严格遵守"只记录明确提到的信息"原则：

| 用户说的话 | ✅ 记录 | ❌ 不推断 |
|-----------|--------|----------|
| "我需要GPU算力" | 需求: GPU算力 | - |
| "我在做AI项目" | - | ❌ 不推断需要GPU |
| "我是产品经理" | 职业: 产品经理 | ❌ 不推断懂用户研究 |
| "我有5年Python经验" | 能力: Python (5年) | - |

**置信度阈值：** 只记录置信度 > 80% 的信号

---

## 🔒 隐私保护

- **本地存储** - 所有档案存储在用户本地
- **用户可控** - 用户决定哪些信息对外可见
- **匿名模式** - 可选择只显示能力，不显示身份
- **可删除** - 用户可随时删除自己的档案

    └── heartbeat_state.json    # 心跳状态

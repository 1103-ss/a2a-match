# A2A Match - SKILL.md

> Agent-to-Agent 智能匹配平台
> 从对话中自动提取需求/能力/资源，零配置匹配

---

## 🎯 核心理念

**不打扰用户，从对话中自然获取信息**

用户在日常对话中会自然透露：
- 职业、公司
- 正在做的项目
- 遇到的困难（需求）
- 擅长的事情（能力）
- 拥有的资源

A2A Match 自动识别并结构化存储，无需用户主动填写。

---

## 💓 心跳机制

A2A Match 通过心跳机制在对话中主动提示匹配机会。

### 触发场景

**当你透露需求时：**
```
用户: 我最近需要找一些GPU算力来训练模型
Agent: 🎯 检测到你的需求：「GPU算力」
       
       我可以帮你找到：
       • 拥有 GPU 资源的 Agent
       • 可以提供算力的用户
       
       要不要看看匹配结果？
```

**当你透露能力时：**
```
用户: 我有5年Python开发经验，做过几个大模型项目
Agent: 💪 检测到你的能力：「Python开发」「AI开发」
       
       我发现有人正在寻找这样的能力！
       要不要看看谁需要你的帮助？
```

**当你透露资源时：**
```
用户: 我有几台服务器闲着，有RTX 4090
Agent: 🎁 检测到你的资源：「RTX 4090 算力」
       
       有人正在寻找这类资源！
       要不要看看匹配的需求方？
```

### 心跳检查内容

1. **检测记忆文件变化**
   - MEMORY.md 更新时重新生成档案

2. **检测匹配机会**
   - 扫描本地缓存的档案
   - 计算匹配度
   - 提示用户

3. **档案过期刷新**
   - 超过24小时自动刷新


## 📝 记忆格式

在 `MEMORY.md` 中添加结构化条目：

```markdown
## A2A Profile

### Needs
- [ ] GPU算力 - 训练大模型 (2026-04-09)
- [ ] 流量渠道 - 推广新产品 (2026-04-08)

### Capabilities
- [x] Python开发 - 5年经验
- [x] AI/大模型 - 参与过3个项目
- [ ] 前端开发 - 熟悉React

### Resources
- [ ] RTX 4090 x2 - 闲置时可共享
```

## ⚠️ 防幻觉原则

### 严格限制

1. **只记录明确提到的信息**
   - ✅ 用户说："我需要GPU算力" → 记录
   - ❌ 用户说："我在做AI" → 推断需要GPU → 不记录

2. **不推断、不联想**
   - ✅ 用户说："我是产品经理" → 记录职业
   - ❌ 用户说："我是产品经理" → 推断懂用户研究 → 不记录

3. **置信度阈值**
   - 只记录置信度 > 80% 的信号
   - 模糊信息需要确认： "你说需要GPU算力，对吗？"

4. **不编造匹配**
   - 找不到匹配 → 直接说"暂无匹配"
   - 不降低标准凑数

## 🔧 配置

### 调整识别灵敏度

在 `a2a/config.json`:

```json
{
  "recognition": {
    "confidence_threshold": 0.8,
    "auto_add": false,
    "confirm_before_add": true
  }
}
```

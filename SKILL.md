## ⚠️ 隐私说明

**本地模式（默认）**：所有数据存本地 `~/.qclaw/workspace/a2a/profile.json`，不上传任何数据，不连接任何人。

**云端模式（匹配需要，用户明确同意后开启）**：上传昵称 + 需求/能力/资源标签用于匹配计算。不上传聊天记录、记忆文件。关闭：说「关闭云端同步」。

## 📂 本地档案格式

`~/.qclaw/workspace/a2a/profile.json`：

```json
{
```

## 🔄 完整产品流程（7步）

```
Step 0 设置虾名   询问"我该怎么称呼你？" → 写入 profile.name
Step 1 触发识别   对话中捕捉需求/能力/资源信号
Step 2 确认写入   用户确认 → 存入 profile.json
Step 3 同步云端   自动调用 POST /api/profile（含 name 字段）
Step 4 云端匹配   服务器匹配引擎计算
Step 5 推送通知   心跳检测到新匹配 → 主动告知
Step 6 接受交流   用户接受 → 互换联系方式
Step 7 即时消息   在 Claw 里直接打字聊天
```

## 🧭 触发机制（AI 必读）

**只在以下两种时机主动触发，不要主动推荐功能。**

### 触发1️⃣：新用户首次对话（档案为空时）

档案 `profile.json` 不存在或为空时，发起引导：

```
🎯 欢迎使用 A2A Match！

这是一个智能供需匹配工具——让「有需求的人」和「有能力的人」互相找到对方。
匹配成功后，你们可以直接在 Claw 里聊天，不用加微信。

首先，我该怎么称呼你？
（这个昵称就是你的「虾名」，唯一标识你）
```

**等用户回答昵称后继续：**

```
好的，{昵称}！

接下来我需要了解你两件事：

【一】你有什么？（能力、资源、渠道、人脉都可以）
【二】你在找什么？（哪怕没想清楚，说个大概方向也行）

随便说几点，比如：
• "我们团队专做小程序开发"
• "我有电商货源，想找分销渠道"
• "我想找GPU算力跑模型"
• "我有4张4090可以共享"
• "我在找合伙人一起做AI项目"
```

**用户回答后确认 → 写入 profile.json → 询问云端同步：**

```
✅ {昵称}，档案已保存！

☁️ 要开启云端同步吗？（这样才能匹配到其他人）
  1. 开启
  2. 先不开，之后再说
```

**开启时**：设 `cloud_config.json` → `cloud.enabled = true` → 调用 `POST /api/profile` → 告知「已同步，等有心跳会自动扫描匹配」。

---

### 触发2️⃣：对话中捕捉到信号

检测到以下关键词时，进入识别流程：

**需求信号**（用户在找什么）："我想找/需要/在找……"、"有没有人能……"、"想合作/对接/认识……"

**能力信号**（用户有什么）："我有/我们能……"、"能提供/可承接……"、"擅长/团队能做……"

**处理**：复述用户说的内容 → 等用户确认 → 写入 profile.json → 同步云端（如已开启）。

## 📋 Step 1~7 完整流程说明

### Step 1 识别 & Step 2 确认

听到信号后复述：「我听到你说：你想找 GPU 算力来训练模型。是这样吗？」

等待用户确认后才记录。用户只说需求时主动追问另一面：「你有什么可以分享的吗？」

### Step 3 写入并同步

用户确认后写入 `profile.json`；若云端已开启则同步到 `POST http://81.70.250.9:3000/api/profile`。

### Step 4 查询匹配

用户说「帮我看看有没有匹配的」时调用：

```
GET http://81.70.250.9:3000/api/matches/{userId}
```

展示匹配卡片：

```
🎉 发现 1 个新匹配！

━━━━━━━━━━━━━━━━━━
【王五 × 你】匹配度 92%
━━━━━━━━━━━━━━━━━━
  匹配原因：你的需求「GPU算力」↔ 对方的资源「RTX 4090 算力」
  对方角色：AI研究员 · 自由职业
━━━━━━━━━━━━━━━━━━

  我想要      跳过
```

### Step 5 心跳检测（HEARTBEAT.md 配置）

每 30 分钟调用 `python scripts/heartbeat_cloud.py check`：

```
无新匹配 → HEARTBEAT_OK
有新匹配 → 展示匹配卡片（格式同上）
有消息   → 💬 收到【{对方}】的消息：「{内容}」
           📝 直接打字回复，我帮你发送
```

### Step 6 接受 / 拒绝

**接受**：`POST /api/match/{matchId}/accept`

- 对方未接受 → 「✅ 意向已表达，等对方回复」
- 双向接受 → 「🎉 匹配成功！可以开始交流了！」（contact_share=true 时显示联系方式）

**拒绝**：`POST /api/match/{matchId}/reject`，无需提示。

**查看联系方式**：`GET /api/match/{matchId}/contact`

### Step 7 即时消息

匹配成功后直接打字回复，AI 自动发送到云端。

**管理多匹配**：用户有多个进行中匹配时，回复前先确认发给谁（说「找{昵称}」切换）。

**查看聊天记录**：用户说「查看消息」「和{昵称}的聊天」时调用：

```
GET http://81.70.250.9:3000/api/match/{matchId}/messages?userId={userId}
```

## ☁️ 云端 API 参考

| 方法 | 端点 | 说明 |
|------|------|------|
| POST | `/api/profile` | 创建/更新档案，触发匹配 |
| GET | `/api/matches/:userId` | 获取我的所有匹配 |
| POST | `/api/match/:id/accept` | 接受匹配 |
| POST | `/api/match/:id/reject` | 拒绝匹配 |
| GET | `/api/match/:id/contact` | 双方接受后获取联系方式 |
| GET | `/api/match/:id/messages` | 获取聊天记录 |
| POST | `/api/message` | 发送消息 |
| GET | `/api/messages/:userId?unread=true` | 获取未读消息 |
| GET | `/api/stats` | 系统统计 |

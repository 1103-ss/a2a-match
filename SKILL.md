# A2A Match - 智能供需匹配平台

> **关键词**: 供需匹配 | 资源对接 | 能力匹配 | 商务合作 | 合伙人 | 需求发布 | 项目合作 | GPU算力 | 技术开发 | 电商货源 | 工厂OEM | 副业 | 找搭子 | 匹配引擎 | 智能撮合 | OPC共创 | AI短剧 | 数字人

---

## 🎯 一句话介绍

**零配置的智能匹配系统** — 从对话中自动识别你的需求/能力，帮你找到合适的合作伙伴，支持云端实时匹配和通知。

---

## ⚠️ 隐私与数据说明（必读）

### 本地模式（默认）
- 所有数据存储在本地 `~/.qclaw/workspace/a2a/` 目录
- **不连接任何云端，不上传任何数据**
- 完全离线可用

### 云端模式（可选，需主动开启）
启用后才连接云端服务器 `http://81.70.250.9:3000`。

**云端同步的数据范围：**
| 数据 | 说明 |
|------|------|
| 你的 UserId | 自动生成（UUID，无个人信息） |
| 昵称 | 你填写的昵称 |
| 标签（Tags） | 从对话中提取的关键词 |
| 需求（Needs） | 你明确说"我需要..."的内容 |
| 资源（Resources） | 你明确说"我有..."的内容 |

**云端不上传的内容：**
- ❌ 你的 MEMORY.md 内容
- ❌ 你的每日记忆文件
- ❌ 任何包含个人隐私的信息

**如何启用/关闭云端：**
编辑 `~/.qclaw/workspace/a2a/cloud_config.json`，将 `cloud.enabled` 改为 `false` 即可关闭。

## 📂 本地档案格式

所有数据存储在 `~/.qclaw/workspace/a2a/profile.json`，结构如下：

```json
{
  "profile": {
    "id": "本地唯一ID（UUID）",
    "name": "你的昵称",
    "role": "你的职业",
    "company": "公司/团队（可选）",
    "industry": "行业",
    "location": "城市（可选）",
    "contact": {
      "email": "你的邮箱",
      "wechat": "你的微信（可选）",
      "preferred": "email 或 wechat"
    },
    "contact_share": false
  },
  "capabilities": [
    {
      "skill": "Python开发",
      "level": "expert",
      "description": "5年Python开发经验"
    }
  ],
  "resources": [
    {
      "type": "gpu",
      "name": "RTX 4090 算力",
      "description": "4卡 RTX 4090，可共享"
    }
  ],
  "needs": [
    {
      "skill": "GPU算力",
      "description": "需要训练大模型",
      "priority": "high"
    }
  ]
}
```

**contact_share 说明：**
- `false`（默认）：匹配被双方接受后，只互换「昵称+职业」，不暴露联系方式
- `true`：匹配被双方接受后，互换「昵称+职业+邮箱/微信」

## 🔄 完整产品流程（6个步骤）

```
Step 1 触发识别   → 对话中捕捉需求/能力信号
Step 2 确认写入   → 用户确认 → 存入 profile.json
Step 3 同步云端   → 自动调用 /api/profile
Step 4 云端匹配   → 服务器匹配引擎计算
Step 5 推送通知   → 心跳检测到新匹配 → 主动告知
Step 6 接受交流   → 用户接受 → 互换联系方式
```

#### 触发1️⃣：新用户首次对话（档案为空时）

当检测到本地档案 `~/.qclaw/workspace/a2a/profile.json` 不存在或为空时，主动发起引导：

```
🎯 欢迎使用 A2A Match！

这是一个智能供需匹配工具——让「有需求的人」和「有能力的人」互相找到对方。

它的工作方式：
• 如果你有什么别人可能需要 → A2A Match 帮你找需要的人
• 如果你在找什么 → A2A Match 帮你扫描有没有合适的

所以我需要了解你两件事：

【一】你有什么？（你的能力、资源、渠道、人脉都可以）
【二】你在找什么？（哪怕没想清楚，说个大概方向也行）

随便说几点就行，比如：
• "我们团队专做小程序开发"
• "我有电商货源，想找分销渠道"
• "我想找GPU算力跑模型"
• "我有4张4090，可以共享"
• "我在找合伙人一起做AI项目"

可以一次性说完，也可以只说一两条，边用边补充。
```

#### 触发2️⃣：对话中捕捉到信号

日常对话中检测到以下关键词，立即进入识别流程：

**需求信号（用户在找什么）：**
- "我想找……" / "我需要……" / "我在找……"
- "有没有人能……" / "谁有……" / "求推荐……"
- "想合作……" / "想对接……" / "想认识……"
- "需要能……的人/团队/资源"
- "想问一下有没有人做……"

**能力/资源信号（用户有什么）：**
- "我有……" / "我们能……" / "我司……"
- "能提供……" / "可承接……" / "可支持……"
- "团队能做……" / "擅长……"
- "手里有……" / "手头有……"

## 📋 Step 1~3：识别 → 确认 → 写入 → 同步

### Step 1：识别提取

用自然语言复述用户说的事实，不推断、不添加：
```
"我听到你说：你想找 GPU 算力来训练模型。是这样吗？"
"我还听到你说：你有 4 张 RTX 4090 闲置可以共享。对吗？"
```

### Step 2：用户确认

等待用户确认后才记录。如果用户只说了需求，没说能力，主动追问另一面：
```
"好的，已记录你的需求：GPU算力。

另外问一下——你现在有什么可以分享的吗？哪怕是一个能力、一个资源、一个渠道都可以。"
```

### Step 3：写入本地档案并同步云端

用户确认后，执行以下操作：

**写入本地 profile.json：**
```python
# 伪代码，AI 按此逻辑操作
写入 profile.json：
  如果是需求 → 加入 needs[]
  如果是能力 → 加入 capabilities[]
  如果是资源 → 加入 resources[]
```

**同步到云端（如果云端已开启）：**
```
调用 POST http://81.70.250.9:3000/api/profile
Body: {
  "userId": profile["profile"]["id"],
  "name": profile["profile"]["name"],
  "email": profile["profile"]["contact"]["email"],
  "tags": ["GPU算力", "小程序开发"],
  "resources": ["RTX 4090 x4"],
  "needs": ["GPU算力（训练模型）"]
}
```

**告知用户：**
```
✅ 已记录！

你的档案已更新 + 已同步到云端匹配池。
下次有心跳检测时会自动扫描有没有合适的。
如果想现在就看，直接说"帮我看看有没有匹配的"。
```

## ✅ Step 6：接受 / 拒绝 / 交流

### 展示匹配卡片

每条匹配单独展示卡片，包含：
- 匹配度分数
- 匹配原因
- 对方昵称 + 职业（不暴露联系方式）
- 两个按钮：「我想要」和「跳过」

### 接受匹配

用户说「我想要」或「接受」时：

**调用 API：**
```
POST http://81.70.250.9:3000/api/match/{matchId}/accept
Body: { "userId": "你的userId" }
```

**处理响应：**

```
情况1：对方尚未接受
  → "✅ 已表达意向。等对方确认后，你们就可以开始交流了。"

情况2：对方已经接受了（双向接受）
  → "🎉 匹配成功！可以开始交流了！"

  contact_share = true 时额外显示：
  📱 对方联系方式
  邮箱：xxx@xxx.com
  微信：xxxxx

情况3：你是第一个接受的
  → "✅ 你的意向已表达。等对方回复后，你们就可以开始交流了。"
```

### 拒绝匹配

用户说「跳过」或「算了」时：
```
POST http://81.70.250.9:3000/api/match/{matchId}/reject
Body: { "userId": "你的userId" }
```
无需额外提示，继续正常对话。

### 主动查看联系方式

用户接受匹配后，可以随时说「怎么联系对方」：
```
GET http://81.70.250.9:3000/api/match/{matchId}/contact
```

```
1. 安装 SkillHub
2. 对 AI 说一句话，触发首次引导
```

## ☁️ 云端 API 参考

| 方法 | 端点 | 说明 |
|------|------|------|
| POST | `/api/profile` | 创建/更新档案，触发匹配 |
| GET | `/api/profile/:userId` | 获取指定用户档案 |
| GET | `/api/matches/:userId` | 获取我的所有匹配 |
| POST | `/api/match/:id/accept` | 接受匹配 |
| POST | `/api/match/:id/reject` | 拒绝匹配 |
| GET | `/api/match/:id/contact` | 双方都接受后，获取联系方式 |
| GET | `/api/stats` | 获取系统统计 |

### WebSocket 事件

```javascript
// 连接
const io = require('socket.io-client');
const client = io('http://81.70.250.9:3000');
client.emit('join', userId);

// 监听新匹配
client.on('new_matches', (data) => {
  // data.matches = 新匹配的列表
  console.log('新匹配：', data);
});

// 监听对方接受了
client.on('match_accepted', (data) => {
  // data.match = 匹配详情
  // 如果双方都接受了，contact 字段包含对方联系方式
  console.log('对方接受了！', data);
});
```
